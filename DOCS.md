# Инструкции для заказчика

## 1. Портфолио: как вставить свои фотографии

Фотографии настраиваются в файле **`src/data/portfolio.js`**.

### Вариант А: фото из папки проекта

1. Создайте папку **`public/portfolio/`** в корне проекта.
2. Положите туда свои изображения (например: `strizhka-1.jpg`, `boroda-2.jpg`).
3. В `src/data/portfolio.js` в массиве `portfolioItems` у каждого элемента замените поле **`image`** на путь к файлу:
   ```js
   image: '/portfolio/strizhka-1.jpg'
   ```
   Слэш в начале означает папку `public` на сайте.

### Вариант Б: фото по ссылке (хостинг, облако)

Если фото лежат в интернете (облако, сайт, CDN), подставьте полную ссылку:
```js
image: 'https://ваш-сайт.ru/foto/rabota-1.jpg'
```

### Остальные поля

- **category** — категория для фильтра: `'hair'` (стрижки), `'beard'` (борода), `'shave'` (бритьё).
- **barber** — имя барбера (подпись).
- **service** — название услуги (подпись).
- **products** — чем пользовались (по желанию).

Можно добавлять и удалять элементы в массиве `portfolioItems` — блок портфолио подстроится.

## 2. Система записей: что передать заказчику и что нужно для работы

Сейчас форма записи **не отправляет данные ни на сервер, ни на почту**. Она только:

- сохраняет выбор в браузере (услуги, барбер, дата/время, контакты);
- показывает экран «Готово! Мы подтвердим запись по SMS».

Чтобы заказчик реально получал заявки и мог ими пользоваться, нужно **подключить бэкенд или внешний сервис**. Ниже — что передать заказчику и какие у него варианты.

### Что передать заказчику

1. **Исходный код проекта** (папка целиком или репозиторий).
2. **Инструкцию по запуску** (ниже).
3. **Этот файл (DOCS.md)** — в нём описаны портфолио и система записей.

### Как заказчику запускать проект

```bash
# Установка зависимостей (один раз)
npm install

# Запуск в режиме разработки
npm run dev

# Сборка для публикации на хостинге
npm run build
```

После `npm run build` папка **`dist/`** — это готовый статический сайт. Её можно залить на любой хостинг (Vercel, Netlify, Beget, Timeweb и т.д.).

### Что нужно, чтобы записи реально доходили до барбершопа

Сейчас при нажатии «Забронировать» данные никуда не уходят. Нужно добавить один из вариантов.

#### Вариант 1: Форма и сервис форм (проще всего)

- Зарегистрировать аккаунт в сервисе форм (например **Formspree**, **Getform**, **Telegram Bot**).
- В форме записи при отправке делать запрос (fetch) на URL этой формы: передать имя, телефон, email, выбранные услуги, барбера, дату и время.
- Заказчик будет получать заявки на почту или в Telegram.

**Что понадобится заказчику:** только аккаунт в выбранном сервисе и подстановка URL в код (один раз может сделать разработчик).

#### Вариант 2: Свой бэкенд (Node.js, PHP и т.д.)

- Написать сервер: принимает данные формы и сохраняет в базу (или в файл) и/или отправляет письмо/SMS.
- В коде формы записи (компонент **Booking**, файл `src/components/Booking.jsx`) в обработчике отправки вызвать `fetch()` на этот сервер вместо текущего только показа «Готово!».

**Что понадобится заказчику:** хостинг с поддержкой Node/PHP и базы (если нужна), либо помощь разработчика для настройки.

#### Вариант 3: Интеграция с системами онлайн-записи

- Подключить API сервиса онлайн-записи (например **Yclients**, **Altegio**, **Booksy** и т.п.).
- В форме подставлять выбранные услуги, мастера, дату и время в запрос к API.
- Запись будет создаваться в кабинете заказчика автоматически.

**Что понадобится заказчику:** договор с выбранным сервисом, API-ключ/токен. Внесение изменений в код (вызов API) обычно делает разработчик.

### Кратко для заказчика

| Задача | Что нужно |
|--------|-----------|
| Просто выложить сайт в интернет | Исходники + `npm run build` → залить папку `dist/` на хостинг. |
| Чтобы клиенты могли «записаться», а заявки приходили на почту/в Telegram | Подключить сервис форм (Formspree и т.п.) или простой бэкенд и один раз прописать вызов в форме (лучше с разработчиком). |
| Чтобы записи попадали в готовую систему учёта (Yclients и т.д.) | Договор с сервисом + интеграция по API (разработчик). |

Текущий код готов к тому, чтобы в нужном месте (отправка формы в **Booking.jsx**) добавить вызов вашего API или URL формы — структура данных (услуги, барбер, дата, время, контакты) уже есть в контексте записи.
